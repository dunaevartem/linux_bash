# del_logs.sh
__del_logs.sh__ — это простой Bash‑скрипт, предназначенный для поиска и удаления файлов логов (.log), которые старше заданного количества дней.
Скрипт делает следующее:

* Просит пользователя указать директорию, в которой будут искаться логи (можно оставить поле пустым, чтобы использовать текущую директорию).
* Проверяет существование указанной директории.
* Запрашивает у пользователя число дней, после которого файлы считаются «старше».
* Находит все файлы с расширением .log, которые старше указанного количества дней, используя команду find.
* Показывает найденный список файлов.
* Спрашивает подтверждение удаления.
* При подтверждении удаляет найденные файлы, иначе завершается без изменений.
# Установка и запуск
* Скопируйте содержимое скрипта в файл del_logs.sh.

* Сделайте его исполняемым:

``` chmod +x del_logs.sh ```
* Запустите скрипт:

``` ./del_logs.sh ```
* Следуйте подсказкам в интерактивном режиме.

Пример работы
Введите директорию для поиска (или нажмите Enter чтобы искать файлы в текущей директории): /var/log
Введите кол-во дней старше которых нужно искать файлы: 30
Найдены .log файлы:
/var/log/app1.log
/var/log/app2.log
Удалить данные файлы? (y/n): y
Файлы удалены
Как это работает
read -p используется для интерактивного ввода.
find "$dir" -type f -name "*.log" -mtime +$days ищет файлы, где -mtime +$days означает «модифицированные более чем $days дней назад».
Файлы сохраняются в массиве files, что позволяет легко итерировать по ним в циклах.
rm -f "$file" беззвучно удаляет каждый найденный файл.
Важные замечания
Будьте осторожны при удалении файлов. Перед запуском убедитесь, что у вас правильный путь и нужное число дней.
Скрипт работает только в среде, где доступен Bash и утилита find.
